# A2UI Playground 功能与布局总结

## 1. 概述

A2UI Playground 是一个用于展示和测试 A2UI 协议的交互式平台，它允许用户通过 Agent 对话或本地模拟流来查看 A2UI 组件的渲染效果。

## 2. 核心功能

### 2.1 Agent 对话系统
- **消息输入与发送**：支持Enter发送消息，Shift+Enter换行
- **消息历史展示**：区分用户和助手消息，显示不同的背景色
- **流式响应**：实时显示模型返回和协议渲染状态
- **仅模型对话模式**：可切换为直接调用 POST /api/chat，不加载 A2UI

### 2.2 A2UI 预览
- **实时渲染**：根据服务端返回的 A2UI 协议实时渲染组件
- **本地模拟流**：支持多种场景的本地模拟数据
- **渲染状态展示**：显示连接、渲染等状态

### 2.3 调试工具
- **Store State 查看**：展示当前组件状态、数据模型和错误信息
- **Errors 查看**：显示解析和渲染过程中的错误
- **A2UI JSON 查看**：
  - 模型原始输出（完整）
  - 下发 JSONL（协议流，与解析器输入一致）
  - 当前 store 反推协议（可选对照）

### 2.4 场景模拟
支持以下本地模拟场景：
- Complex Nested Tree
- Row and Column Mixed
- Card Demo
- Data binding
- List + template
- Shopping cart list
- Local action → dataModel
- Agent back（测试列+文案+按钮）

## 3. 布局结构

### 3.1 整体布局
- **左侧边栏**：Agent 对话区域（宽度420px）
- **右侧主区域**：A2UI 预览区域（自适应宽度）

### 3.2 左侧边栏
1. **顶部控制区**：
   - 标题："Agent 对话"
   - 模式提示信息
   - "仅模型对话"开关

2. **场景选择区**：
   - 标题："本地模拟流 · 场景"
   - 场景选择下拉框
   - 模式说明文本

3. **消息历史区**：
   - 空状态提示
   - 消息列表（用户/助手消息）
   - 助手消息包含：
     - 加载状态（模型返回中、协议渲染中）
     - 完成状态标签（已完成/未正常完成）
     - 模型原始输出折叠面板（调试用）

4. **消息输入区**：
   - 文本输入框（支持自动调整高度）
   - 发送按钮

### 3.3 右侧主区域
1. **顶部控制栏**：
   - 标题："A2UI Playground"
   - 操作按钮：
     - View Store
     - View Errors
     - View A2UI JSON
     - 本地模拟流

2. **预览卡片**：
   - 标题栏（显示当前模式和加载状态）
   - 预览内容区：
     - 仅模型对话模式：显示模式说明
     - A2UI 模式：显示渲染的组件

### 3.4 模态框
1. **Store State 模态框**：
   - 组件总数统计
   - 数据模型展示
   - 完整 store 状态 JSON

2. **Errors 模态框**：
   - 错误列表（类型和描述）

3. **A2UI JSON 模态框**：
   - 模型返回（完整）
   - 下发 JSONL（协议流）
   - 当前 store 反推协议

## 4. 样式设计

### 4.1 色彩方案
- **背景色**：浅灰色 (#f5f5f5) 主背景，白色 (#fff) 卡片背景
- **消息气泡**：
  - 用户消息：浅蓝色 (#e6f4ff)
  - 助手消息：浅灰色 (#fafafa)
- **状态标签**：
  - 成功：绿色
  - 错误：红色
- **预览区**：白色背景，虚线边框

### 4.2 布局样式
- **响应式设计**：支持不同屏幕尺寸
- **弹性布局**：使用 flex 布局实现自适应
- **间距**：统一的内边距和外边距
- **边框**：使用浅灰色边框分隔区域

### 4.3 组件样式
- **卡片**：使用 Ant Design 卡片组件，带有阴影和圆角
- **按钮**：主要操作使用蓝色主按钮，次要操作使用默认按钮
- **输入框**：带有 placeholder 和自动调整高度
- **加载状态**：使用 Ant Design 的 Spin 组件
- **折叠面板**：用于展示模型原始输出

## 5. 技术实现

### 5.1 核心技术栈
- **前端**：React + TypeScript
- **UI 库**：Ant Design
- **状态管理**：A2UI Store
- **协议解析**：A2UI Parser
- **流式处理**：Server-Sent Events (SSE)

### 5.2 数据流
1. **用户输入** → **发送请求** → **服务端处理** → **SSE 流式返回** → **解析协议** → **渲染组件**
2. **本地模拟** → **选择场景** → **加载 mock 数据** → **模拟流式传输** → **解析协议** → **渲染组件**

### 5.3 关键功能实现
- **SSE 处理**：使用 `consumeAgentSse` 和 `consumeChatSse` 函数处理流式响应
- **协议解析**：使用 A2UI Parser 解析 JSONL 格式的协议
- **组件渲染**：使用 A2UI React 渲染器渲染组件树
- **状态管理**：使用 A2UI Store 管理组件状态和数据模型
- **模拟流**：使用 `StreamSimulator` 模拟流式传输

## 6. 使用指南

### 6.1 基本操作
1. **启动服务**：运行 `pnpm dev:server` 启动服务端（默认端口 3847）
2. **选择模式**：
   - 标准模式：通过 /api/agent 流式加载 A2UI
   - 仅模型对话：直接调用 /api/chat
3. **选择场景**：在本地模拟流模式下选择不同的场景
4. **发送消息**：在输入框中输入消息并按 Enter 发送
5. **查看预览**：在右侧查看 A2UI 组件渲染效果

### 6.2 调试功能
1. **查看 Store**：点击 "View Store" 按钮查看当前组件状态
2. **查看 Errors**：点击 "View Errors" 按钮查看错误信息
3. **查看 A2UI JSON**：点击 "View A2UI JSON" 按钮查看协议详情
4. **本地模拟流**：点击 "本地模拟流" 按钮使用本地 mock 数据

### 6.3 开发调试
- **开启调试日志**：在浏览器控制台执行 `localStorage.setItem('a2uiAgentDebug','1')` 后刷新
- **查看网络请求**：在浏览器开发者工具中查看 /api/agent 和 /api/chat 请求
- **检查组件结构**：使用 React DevTools 查看组件树

## 7. 总结

A2UI Playground 是一个功能完整、界面友好的 A2UI 协议测试平台，它提供了以下核心价值：

1. **直观展示**：实时渲染 A2UI 组件，直观展示协议效果
2. **多场景测试**：支持多种本地模拟场景，方便测试不同类型的组件
3. **完整调试工具**：提供 Store、Errors 和 JSON 查看功能，方便调试
4. **流式体验**：模拟真实的流式传输，展示 A2UI 的实时渲染能力
5. **易于使用**：简洁的界面设计和直观的操作流程

通过 A2UI Playground，开发者可以快速了解 A2UI 协议的工作原理，测试不同场景下的组件渲染效果，为 A2UI 应用的开发和调试提供有力支持。
